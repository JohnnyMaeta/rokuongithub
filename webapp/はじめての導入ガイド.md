# 🎙️ 録音くん（Webアプリ版）はじめての導入ガイド

> **このガイドの対象**: プログラミング・サーバー運用は初めてだけど、録音くんを自分の学校で使ってみたい先生向け。専門用語は最小限・全工程「クリックする場所」までガイドします。

---

## このアプリは何ができるの？

子どもたちがブラウザで録音すると、**先生のOneDrive（マイクロソフトのクラウド）に自動で保存される**Webアプリです。

- 子どもは **サインイン不要**（URLにアクセスするだけ）
- 先生が登録した「保存先フォルダ」に整理して保存される
- 録音時間制限なし
- スマホ・タブレット・PC どれでもOK

---

## 必要なもの（事前チェック）

導入を始める前に、以下が**全部揃っているか**確認してください：

- [ ] **Microsoftアカウント**（OneDriveが使えるアカウント）
  - 学校で配布された Office 365 のアカウントでもOK
  - 個人の Outlook.com / Hotmail でもOK
- [ ] **GitHubアカウント**（無料）
  - 持っていなければ <https://github.com/signup> で作る（メアド・パスワードを決めるだけ・1分）
- [ ] **クレジットカード不要**（VercelもAzureも無料枠で使えます）
- [ ] **作業時間**: 約30〜45分（初めての場合）
- [ ] **PC**（できればChromeかEdge推奨）

---

## 全体の流れ（地図）

```
[1] Microsoft Entra ID（Azure）でアプリ登録    … 10分
       ↓
[2] このリポジトリを自分のGitHubにコピー（Fork）  … 1分
       ↓

       ↓
[8] 子どもたちにURLを共有 🎉
```

# ステップ 1：Microsoft Entra ID でアプリ登録（10分）

> **何をしているの？**: 「録音くんから先生のOneDriveに書き込む権利」をMicrosoftにお願いする手続きです。

### 1-1. Azure Portal にアクセス

ブラウザで <https://portal.azure.com/> を開く → 自分のMicrosoftアカウントでサインイン。

### 1-2. Microsoft Entra ID を開く

画面上部の検索バーに **「Microsoft Entra ID」** と入力 → 出てきたものをクリック。

### 1-3. 新しいアプリを登録

1. 左側のメニューから **「アプリの登録」** をクリック
2. 上の方の **「+ 新規登録」** をクリック
3. 以下のように入力：

| 項目 | 入力する値 |
|---|---|
| **名前** | `録音くん`（何でもOK・自分が分かれば良い） |
| **サポートされているアカウントの種類** | **「任意の組織のディレクトリ内のアカウント＋個人のMicrosoftアカウント」** を選ぶ |
| **リダイレクトURI** | （**まだ何も入れない**・後で設定します） |

4. 一番下の **「登録」** をクリック

### 1-4. 「クライアントID」をメモ

登録完了画面で表示される：
- **アプリケーション (クライアント) ID** : `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`

これを **メモ帳にコピペして保存**しておいてください。後で **`MICROSOFT_CLIENT_ID`** として使います。

### 1-5. 「クライアントシークレット」を作る

1. 左メニュー **「証明書とシークレット」** をクリック
2. **「+ 新しいクライアント シークレット」** をクリック
3. 入力：
   - **説明**: `Vercel`（何でもOK）
   - **有効期限**: **「24か月」** を選ぶ
4. **「追加」** をクリック

### ⚠️ ここが**最重要ポイント**

クリック後、画面に「**値**」と「シークレットID」が表示されます。

- **「値」** の方を**今すぐコピー**してメモ帳に保存！
- ⚠️ ページを離れると **二度と表示されません**（再表示できない）
- これが **`MICROSOFT_CLIENT_SECRET`** として使う値です

> **もしメモし忘れたら**: 同じページで **「+ 新しいクライアント シークレット」** をもう一度クリックして作り直してください（古いのは「削除」してOK）。

### 1-6. 必要な権限を追加

1. 左メニュー **「APIのアクセス許可」** をクリック
2. **「+ アクセス許可の追加」** をクリック
3. **「Microsoft Graph」** をクリック
4. **「委任されたアクセス許可」** をクリック
5. 検索ボックスに `Files.ReadWrite` と入力 → チェックを入れる
6. 検索ボックスをクリアして `User.Read` と入力 → チェックを入れる
7. 検索ボックスをクリアして `offline_access` と入力 → チェックを入れる
8. 一番下の **「アクセス許可の追加」** をクリック

### ✅ ステップ1 完了チェック

- [ ] メモ帳に「**クライアントID**」を保存した
- [ ] メモ帳に「**シークレットの値**」を保存した
- [ ] 3つの権限（Files.ReadWrite / User.Read / offline_access）を追加した

# ステップ 3：Vercelにデプロイ（5分）

> **何をしているの？**: アプリをインターネット上で動くようにします。

### 3-1. Vercelに登録

1. <https://vercel.com/signup> を開く
2. **「Continue with GitHub」** をクリック
3. GitHubの認証画面が出たら許可
4. 「Hobby」プラン（無料）を選ぶ
5. 名前などを聞かれたら適当に入力

### 3-2. プロジェクトを作る

1. ダッシュボードで右上の **「Add New...」** → **「Project」** をクリック
2. リポジトリ一覧に `rokuongithub` があるはず → 右の **「Import」** をクリック
   - **見つからない場合**: 「Adjust GitHub App Permissions」をクリックして、自分の `rokuongithub` リポジトリへのアクセスを許可

### 3-3. 設定を入力（**ここが最重要**）

「**New Project**」画面が開きます。以下の通り設定：

#### Root Directory（**忘れると失敗します**）

- **Root Directory** の右にある **「Edit」** をクリック
- 一覧から `webapp` をダブルクリック（または選んで「Continue」）
- ボックスに `webapp` と表示されればOK

#### Environment Variables（環境変数）

「Environment Variables」の見出しをクリックして展開。

**1つ目を入力**：
- **Key** 欄（`EXAMPLE_NAME` と薄く表示されている所）に `MICROSOFT_CLIENT_ID` と入力
- **Value** 欄に、ステップ1-4でメモした **クライアントID** を貼り付け

**2つ目を追加**：
- 右下の **「+ Add More」** をクリック → 新しい入力欄が出る
- **Key**: `MICROSOFT_CLIENT_SECRET`
- **Value**: ステップ1-5でメモした **シークレットの「値」** を貼り付け

### 3-4. デプロイ実行

画面一番下の黒い **「Deploy」** ボタンをクリック。

→ 「🎉 Congratulations」と花火が表示されたら成功！1〜2分かかります。

### 3-5. URLをメモ

デプロイ完了画面で表示されるURL（例：`https://rokuongithub.vercel.app/`）を**メモ帳に保存**。

> ※ プロジェクト名によって URL は異なります。あなたの場合は `https://<プロジェクト名>.vercel.app/` です。

### ✅ ステップ3 完了チェック

- [ ] Root Directory に `webapp` を入れた
- [ ] 環境変数を **2つ** 追加した
- [ ] デプロイが「Ready」状態になった
- [ ] 自分のVercel URLをメモ帳に保存した

# ステップ 4：Upstash for Redis を接続（5分）

> **何をしているの？**: 先生のサインイン情報や保存先フォルダ名のリストを入れる「金庫」を作って、アプリにつなぎます。

### 4-1. Storage タブを開く

1. プロジェクト画面（rokuongithub）の **左サイドバー** から **「Storage」** をクリック
   - ⚠️ 上部のタブではなく **左の縦メニュー**にあります（「Domains」の下、「Flags」の上）

### 4-2. データベースを作る

1. 「Create a database」 という画面になる
2. 下にスクロールして **「Marketplace Database Providers」** という見出しを探す
3. その中の **「Upstash」**（説明: `Serverless DB (Redis, Vector, Queue, Search)`）の行を**クリックして展開**
4. 展開された中の **「Upstash for Redis」**（赤い丸ロゴ）の右の **「Create」** ボタンをクリック
   - ⚠️ 「Upstash Vector」「Upstash QStash」「Upstash Search」は別物なので**選ばない**

### 4-3. データベース設定

1. **Database Name**: `rokuon-kv`（何でもOK）
2. **Primary Region**: `Tokyo (ap-northeast-1)` を選ぶ（一番近いから速い）
3. **Plan**: **Free** を選ぶ
4. **Continue** をクリック
5. 利用規約の画面が出たら同意して **Create**

### 4-4. プロジェクトに接続

作成完了後、右上の **「Connect to Project」** ボタンが見えます。

#### パターンA：すでに接続済みの場合（よくある）

`Connect to Project` を押すと、`rokuongithub` の右に青い **「Connected」** バッジが付いていてグレーアウト。
→ **「Cancel」** で閉じればOK（既に接続済み）。

#### パターンB：まだ接続されていない場合

1. **「Connect to Project」** をクリック
2. `rokuongithub` を選ぶ
3. Environments は **Production / Preview / Development** すべてチェック
4. **「Connect」** をクリック

### 4-5. 再デプロイ（**忘れると動きません**）

環境変数を追加したので、もう一度デプロイし直す必要があります。

1. 左サイドバーの **「Deployments」** をクリック
   - ⚠️ ここも上部タブではなく**左サイドバー**
2. 一番上の Production デプロイの右端 **「⋯」**（3点メニュー） → **「Redeploy」**
3. 確認ダイアログ：
   - 「Use existing Build Cache」のチェックは **外したまま**
   - **「Redeploy」** をクリック
4. 1〜2分待つ → ステータスが **「Ready」** になれば完了

### ✅ ステップ4 完了チェック

- [ ] Upstash for Redis を作成した
- [ ] プロジェクトに接続した（または「Connected」バッジ確認）
- [ ] 再デプロイが「Ready」になった

# ステップ 5：Microsoft Entra に「戻り先URL」を登録（2分）

> **何をしているの？**: ステップ1で作ったアプリに、「サインイン後、ここに戻ってきて」と教えます。

### 5-1. Azure Portal に戻る

ブラウザで <https://portal.azure.com/> を開く → ステップ1で作った **「録音くん」** アプリを開く。

> **見つからない時**: ホーム → Microsoft Entra ID → アプリの登録 → 一覧から「録音くん」をクリック

### 5-2. 認証ページを開く

左メニューから **「Authentication (Preview)」**（または「認証」）をクリック。

### 5-3. リダイレクトURIを追加

1. 画面中央の青いボタン **「+ リダイレクト URI の追加」** をクリック
2. 右側にパネルが開く
3. プラットフォームを聞かれたら **「Web」** を選ぶ
4. **リダイレクト URI** 欄に、以下を**正確に**入力：

```
https://<あなたのVercel URL>/api/auth-callback
```

例：
```
https://rokuongithub.vercel.app/api/auth-callback
```

> ⚠️ 末尾の `/api/auth-callback` を忘れずに！スラッシュも含めて1文字も間違えないように。

5. URLの右に緑のチェック ✅ が付けば形式OK
6. 下の「アクセストークン」「IDトークン」のチェックボックスは **触らない**（OFFのまま）
7. パネル下部の青い **「構成」** ボタンをクリック

### ✅ ステップ5 完了チェック

- [ ] リダイレクトURI一覧にあなたのVercel URLが追加された

# 🔧 よくある困った！とその解決法

### Q1. デプロイがビルド失敗する

**原因**: Root Directory に `webapp` が設定されていない。

**対処**:
1. Vercel → プロジェクト → **Settings**（左サイドバー）
2. **General** → **Root Directory**
3. **Edit** をクリックして `webapp` を選択
4. **Save**
5. Deployments → ⋯ → Redeploy

---

### Q2. サインイン時に `redirect_uri_mismatch` エラー

**原因**: ステップ5のリダイレクトURIが正しく設定されていない。

**対処**:
1. Azure → 録音くんアプリ → Authentication (Preview)
2. リダイレクトURIが `https://<あなたのURL>/api/auth-callback` で**完全一致**しているか確認
3. 末尾のスラッシュ・大文字小文字・タイポをチェック

---

### Q3. 子どもの画面で「先生がまだグループを登録していません」

**原因**: ステップ7をやっていない、または保存に失敗した。

**対処**: `/admin.html` を開いて、② のテキストエリアに入力 → 保存。

---

### Q4. 子どもの画面で「管理者がまだサインインしていません」

**原因**: ステップ6をやっていない、またはトークンが切れた（90日無使用）。

**対処**: `/admin.html` でもう一度サインイン。

---

### Q5. アップロードで「グループが選択されていない…」エラー

**原因**: 子どもがブラウザの戻るボタンや古いキャッシュを使ったため、登録されていないフォルダ名で送信された。

**対処**: 子ども側のブラウザを **再読み込み（リロード）** する（Cmd+R / Ctrl+R）。

---

### Q6. しばらく使わなかったら動かなくなった

**原因**: リフレッシュトークンが90日無使用で失効した。

**対処**: `/admin.html` で再サインイン。

---

### Q7. コードを変更したのにVercelが反映してくれない

**原因**: ローカルからpushしたリポジトリと、Vercelが見ているリポジトリが違う（fork運用の場合）。

**対処**:
1. GitHubで自分のfork（`<あなた>/rokuongithub`）を開く
2. ファイル一覧の上のバナーの **「Sync fork」** → **「Update branch」** をクリック
3. Vercelが自動で再デプロイ開始

---

### Q8. Vercelの画面に「KV」がない

**正常です**。Vercel KV は廃止され、いまは Marketplace の「Upstash for Redis」に置き換わっています。ステップ4の手順通りにやってください。

---

### Q9. マイクが使えない・録音できない

**原因**:
- ブラウザでマイクが許可されていない
- HTTPS接続でない（VercelはHTTPSなのでOKのはず）

**対処**:
- ブラウザのアドレスバー左の鍵マーク → サイトの設定 → マイクを「許可」
- iOS Safari の場合：設定 → Safari → マイク → 許可

---

# 🛠 メンテナンス

### 保存先フォルダ名を増やす・減らす

`/admin.html` を開いて、テキストエリアを編集 → 「保存」。
→ **再デプロイは不要**。子どもが画面を再読み込みすれば反映される。

### 別のMicrosoftアカウントに切り替えたい

そのまま `/admin.html` で別アカウントでサインインすれば上書きされます。

### コードを更新したい

ローカルでコード編集 → push → fork で「Sync fork」→ Vercel自動デプロイ。
